## Role Format Tips

<Callout type="info">
**Role Format Options:**
- **@RoleName** — Use @ symbol (e.g., `🎮:@Gamer`)
- **RoleName** — Use role name directly (e.g., `🎮:Gamer`)
- **"Role Name"** — Use quotes for spaces (e.g., `💬:"Gaming Role"`)
- **[@Role1, @Role2]** — Create inline multi-role arrays (e.g., `💡:[@Mod, @Admin]`)
- **[BundleName]** — Import a saved Role Bundle (e.g., `🚀:[StarterPack]`)
- **emoji:role:limit** — Set a maximum number of users per role (e.g., `🎮:Gamer:10`)

**Examples:**

- `🎮:@Gamer,🎨:@Artist` — Using @ symbol
- `🎮:Gamer,🎨:Artist` — Direct role names
- `🎮:"Gaming Role",🎨:"Art Role"` — Quoted names with spaces
- `💡:[@Moderator, @Admin]` — Inline multi-role array
- `🚀:[StarterPack]` — Import a Role Bundle
- `🎮:Gamer:10,🎨:Artist:5` — With user limits

</Callout>

## Troubleshooting

<Callout type="warning">
  - **Bot not posting message:** Verify bot permissions (Manage Roles, Send
  Messages, Add Reactions).
  <br />
  - **Roles not assigned:** Ensure the bot's role is positioned above the target
  roles in the Discord role list.
  <br />
  - **Emoji not working:** Use standard emoji or custom emoji accessible to the
  bot.
  <br />- **Message not found:** Verify the message ID is correct and the
  message still exists.

</Callout>

## Best Practices

<Callout type="success">
  - Maintain manageable role lists (5-12 per message is recommended).
  <br />
  - Use clear, descriptive role names and appropriate emojis.
  <br />
  - Test your setup with a test user before deployment.
  <br />
  - Regularly clean up unused role-reaction messages.
  <br />- Use the update command to modify existing messages instead of creating
  new ones.
