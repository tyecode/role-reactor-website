## Role Format Tips
<Callout type="info">
**Role Format Options:**
- **@RoleName** - Use @ symbol (e.g., `@Gamer`)
- **RoleName** - Use role name directly (e.g., `Gamer`)
- **Role Mention** - Use role mention format (e.g., `&lt;@&123456789&gt;`)
- **"Role Name"** - Use quotes for spaces (e.g., `"Gaming Role"`)

**Examples:**
- `ðŸŽ®:@Gamer,ðŸŽ¨:@Artist` - Using @ symbol
- `ðŸŽ®:Gamer,ðŸŽ¨:Artist` - Direct role names
- `ðŸŽ®:"Gaming Role",ðŸŽ¨:"Art Role"` - Quoted names with spaces
</Callout>

## Troubleshooting
<Callout type="warning">
- **Bot not posting message:** Verify bot permissions (Manage Roles, Send Messages, Add Reactions).<br />
- **Roles not assigned:** Ensure the bot's role is positioned above the target roles in the Discord role list.<br />
- **Emoji not working:** Use standard emoji or custom emoji accessible to the bot.<br />
- **Message not found:** Verify the message ID is correct and the message still exists.
</Callout>

## Best Practices
<Callout type="success">
- Maintain manageable role lists (5-12 per message is recommended).<br />
- Use clear, descriptive role names and appropriate emojis.<br />
- Test your setup with a test user before deployment.<br />
- Regularly clean up unused role-reaction messages.<br />
- Use the update command to modify existing messages instead of creating new ones.
</Callout>
