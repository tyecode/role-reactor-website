## Quick Installation

<Callout type="info">
**Recommended Method:** Use the pre-configured invitation link with all required permissions included.
</Callout>

<Steps>
<Step>

### Access the Invitation Link

[**Add Role Reactor Bot to Your Server**](https://discord.com/api/oauth2/authorize?client_id=1392714201558159431&scope=bot+applications.commands&permissions=268790880)

The invitation link includes all necessary permissions pre-selected for optimal functionality.

</Step>
<Step>

### Select Target Server

1. Choose your server from the dropdown menu
2. Verify you have "Manage Server" permission (required to add bots)
3. Click "Continue" to proceed

</Step>
<Step>

### Review Required Permissions

The bot requires the following permissions to function properly:

- **Manage Roles** - Assign and remove roles from members
- **Manage Messages** - Manage role-reaction messages
- **Add Reactions** - Add emoji reactions to role selection messages
- **Read Message History** - Monitor reaction events and read channel history
- **View Channel** - Access channels where the bot operates
- **Send Messages** - Respond to commands and create role messages
- **Embed Links** - Create rich embedded messages
- **Manage Server** - Manage server settings and configurations
- **Use External Emojis** - Use emojis from other servers

Click "Authorize" to grant these permissions and complete the installation.

</Step>
<Step>

### Verify Installation

1. Check your server member list - Role Reactor Bot should appear online
2. Verify the bot is visible in your channels
3. Test functionality by running `/help` to confirm the bot responds

</Step>
</Steps>

## Manual Permission Configuration

If you prefer to configure permissions manually or need to adjust them after installation:

### Required Permissions

Ensure Role Reactor Bot has the following permissions in your server:

- **Manage Roles** - Essential for role assignment operations
- **Manage Messages** - Required to manage role-reaction messages
- **Add Reactions** - Necessary to add emojis to role messages
- **Read Message History** - Required to detect member reactions and read channel history
- **View Channel** - Needed to access channels where the bot operates
- **Send Messages** - Required to respond to commands and create role messages
- **Embed Links** - Required to create rich embedded messages
- **Manage Server** - Required to manage server settings and configurations
- **Use External Emojis** - Required to use emojis from other servers

### Optional Permissions

For advanced features, the bot may also benefit from:

- **Move Members** - Required for voice channel disconnection (voice restrictions)
- **Mute Members** - Required for voice channel muting (voice restrictions)

### Permission Value

If you need the permission value for manual configuration: **268790880**

## Role Hierarchy Configuration

<Callout type="warn">
**Critical:** The bot can only manage roles that are positioned below its own role in your server's role hierarchy.
</Callout>

<Steps>
<Step>

### Access Server Settings

1. Right-click your server name
2. Select "Server Settings"
3. Navigate to "Roles" in the left sidebar

</Step>
<Step>

### Configure Bot Role Position

1. Locate "Role Reactor Bot" role in the role list
2. Drag the bot's role upward so it is positioned above any roles you want it to manage
3. Save your changes

</Step>
</Steps>

### Recommended Role Hierarchy

```
@Server Owner (highest position)
@Administrators
@Moderators
@Role Reactor Bot  ← Bot role positioned here
@VIP Members       ← Bot can manage this
@Gamers           ← Bot can manage this
@Artists          ← Bot can manage this
@everyone (lowest position)
```

## Optional Customization

### Bot Nickname

To customize the bot's display name:

1. Right-click the bot in your member list
2. Select "Change Nickname"
3. Enter your preferred name (e.g., "Role Manager", "Roles")

### Custom Bot Role

To create a custom role for the bot:

1. Navigate to Server Settings > Roles
2. Create a new role with your preferred name and color
3. Assign the role to the bot
4. Position the role correctly in the hierarchy
